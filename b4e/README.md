# BERT4ETH (B4E) — Ethereum Phishing Detection Extension

This folder contains the **extension experiment** applying AnomalyGFM to a new, unseen dataset — the BERT4ETH Ethereum transaction graph for phishing account detection.

Unlike the 11 benchmark datasets (which already come as attributed graphs), BERT4ETH consists of raw **Ethereum transaction logs** (tabular data). A complete graph construction pipeline was built from scratch to transform it into the same format as T-Finance/T-Social, then fed into the **same pre-trained large-scale inference model** without any retraining.

## 🚀 How to Run

### Step 0 — Extract the Dataset

The 4 raw CSV files are too large to commit directly (~6 GB total). They are archived in `data_raw/b4e_dataset.rar`.

|---|---|---|

| `normal_eoa_transaction_out_slice_1000K.csv` | ~5 GB | Outgoing transactions of 1M normal accounts |
| `phisher_transaction_in.csv` | ~100 MB | Incoming transactions of known phishing accounts |

Extract into `b4e/data_raw/`:

```bash
cd b4e/data_raw
7z x b4e_dataset.rar -o./       # 7-Zip
# or: right-click → Extract Here (WinRAR)
```

After extraction, `data_raw/` should contain:

```
data_raw/
├── b4e_dataset.rar
├── phisher_account.txt
├── normal_eoa_transaction_in_slice_1000K.csv
├── normal_eoa_transaction_out_slice_1000K.csv
├── phisher_transaction_in.csv
└── phisher_transaction_out.csv
```

### Step 1 — Run Preprocessing

Open and **Run All** cells in `b4e/src/preprocess.ipynb`.

This generates all files in `data_structured/` automatically:

| Order | Function | Input | Output |
|---|---|---|---|
| 1 | `load_data()` | 4 CSV files from `data_raw/` | In-memory transaction dicts |
| 2 | `seq_duplicate()` | Transaction dicts | Aggregated sequences (3-day window merge) |
| 3 | `generate_bitcoin_mat()` | Aggregated sequences + `phisher_account.txt` | `data_structured/b4e.mat` |
| 4 | `generate_svd_mat()` | `data_structured/b4e.mat` | `data_structured/b4e_svd_10.mat` |
| 5 | `generate_npy_for_fast_inference()` | `data_structured/b4e_svd_10.mat` | `data_structured/b4e_feature_5_1.npy` + `b4e_label_5_1.npy` |

### Step 2 — Run Inference

```bash
cd b4e/src
python fast_inference_main.py
```

Outputs: **AUROC**, **AUPRC**, **Precision@0.1%**, **Recall@0.1%**, **Precision@0.05%**, **Recall@0.05%**

## 🔧 Pipeline Details

### Graph Construction (`generate_bitcoin_mat`)

Since BERT4ETH is tabular data, a full graph must be built:

1. **Node discovery** — all unique Ethereum addresses (both normal and phishing)
2. **Edge construction** — directed edges `sender → receiver` for each transaction
3. **Feature extraction** — 28 handcrafted features per node (see below)
4. **Preprocessing** — `log1p` on skewed columns, signed-log on balance, `RobustScaler`, clip `[-5, 5]`
5. **Label mapping** — match addresses against `phisher_account.txt` → `Label: 0/1`

### 28 Node Features

| Category | Features (9) |
|---|---|
| **Volume** | `total_val_in/out`, `avg_val_in/out`, `avg_tx_val_in/out`, `balance`, `turnover_rate`, `net_flow_ratio` |
| **Count & Degree** | `total_tx_in/out`, `unique_src/dst_addresses`, `tx_per_address`, `in_out_addr_ratio`, `addr_flow_ratio`, `addr_reuse_rate` |
| **Temporal** | `lifespan_days`, `avg_hours_between_tx`, `std_hours_between_tx`, `tx_per_hour`, `overall_val_velocity` |
| **Aggregation** | `total_clusters_in/out`, `tx_per_clusters`, `avg_clusters_in/out`, `max_clusters_size` |

> *"Clusters"* = groups of transactions between the same address pair within a 3-day window (merged by `seq_duplicate()`).

### SVD Preprocessing (`generate_svd_mat`)

Reduces features from 28-dim → 10-dim (same as all other datasets: unified `d' = 10`).  
A second `RobustScaler` is applied **after** SVD for numerical stability.

### Subgraph Sampling (`generate_npy_for_fast_inference`)

For large-scale inference (same approach as T-Finance/T-Social):

- DGL random walk (length = `2×S`) per node → sample `S-1` neighbors
- Target node always placed at position `[:, -1, :]` in the subgraph tensor
- Output shape: `[N, S=5, D=10]`
- Isolated nodes: self-padded with their own features

### Inference (`fast_inference_main.py`)

- Loads `data_structured/b4e_feature_5_1.npy` + `b4e_label_5_1.npy`
- Reuses `model_residual2.pth` (pre-trained on zero-shot benchmark — **no retraining**)
- Uses **identity adjacency matrix** → nodes in each subgraph processed independently
- `BETA = 4.0` (heterophilous graph: phishing nodes behave differently from neighbors)
- Batch size: 32768

---

## ⚙️ Key Hyperparameters

| Parameter | Value | Reason |
|---|---|---|
| `SUBGRAPH_SIZE` (S) | 4 | Subgraph size = 4 neighbors + 1 target |
| `emb_dimension` (d') | 10 | Unified SVD dimension across all datasets |
| `BETA` | 4.0 | Heterophilous graph (phishers ≠ their neighbors) |
| `BATCH` | 32768 | GPU memory-efficient batched inference |
| 3-day merge window | 259200 sec | Aggregate repeated transactions between same pair |

## 📊 Evaluation Metrics

Because phishing accounts are extremely rare, standard AUROC/AUPRC are supplemented with top-K precision/recall:

| Metric | Description |
|---|---|
| **AUROC** | Area under ROC curve |
| **AUPRC** | Area under Precision-Recall curve |
| **Precision@0.1%** | Fraction of true phishers in top 0.1% scored nodes |
| **Recall@0.1%** | Fraction of all phishers captured in top 0.1% |
| **Precision@0.05%** | Same at top 0.05% cutoff |
| **Recall@0.05%** | Same at top 0.05% cutoff |
